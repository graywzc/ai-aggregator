# Project Instructions

When starting implementation work, create a fresh git worktree under `.codex/worktrees/<some-task>` on a new `codex/` branch based on `main`, unless the user explicitly says not to.

The branch name and worktree folder should clearly summarize the work. Use a short, descriptive slug such as:

```text
codex/update-auth-flow
.codex/worktrees/update-auth-flow
```

Work only inside the new worktree for that task.
